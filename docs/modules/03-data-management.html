Module 3: Data Management

Plan, organize, and publish your research data

🎯 Learning Objectives

After completing this module, you will be able to:

  • Explain what a Data Management Plan (DMP) is and why it is required
  • Use DMPTool to write a DMP for your research
  • Apply best practices for organizing, naming, and documenting data files
  • Understand the difference between open and proprietary file formats
  • Publish your data with Dryad and increase your research impact

Overview

This module introduces data management plans — an integral component of your research — and data publishing. A DMP is required for IRB applications and grant applications. Publishing your data in an open access repository increases your publication count and supports data reuse and research validation.

✅ Complete the assignment at the end of this module to earn credit toward your certificate!

What Is a Data Management Plan?

A Data Management Plan (DMP) is a formal document that describes:

  • What data you will collect or generate
  • How the data will be organized, stored, and backed up
  • What formats and standards you will use
  • Who will have access to the data during and after the project
  • How and where the data will be shared or preserved after the project ends
WarningWhen Are DMPs Required?
  • IRB (Institutional Review Board) applications at UCR
  • Federal grant applications (NSF, NIH, NEH, and most other funders)
  • Many journal and publisher submission requirements

DMPTool

DMPTool is a free online tool that helps researchers write data management plans. It provides:

  • Funder-specific DMP templates (NSF, NIH, NEH, etc.)
  • Guidance text for each section
  • Collaboration features to share drafts with your advisor
  • Example DMPs from other researchers

Data Organization Best Practices

File Naming Conventions

Use consistent, descriptive file names that include:

  • Project identifier
  • Date (YYYY-MM-DD format)
  • Version number

Example: smith-dissertation-survey-data-2025-01-15-v2.csv

Folder Structure

Organize files into logical folders:

project/
  ├── raw-data/          # Never modify original data
  ├── processed-data/    # Cleaned, analysis-ready data
  ├── scripts/           # Analysis code
  ├── outputs/           # Figures, tables, reports
  └── docs/              # README, codebook, DMP

Documentation Files

Document Purpose
README High-level overview of the project and instructions for using the files
Codebook Defines specific variables, units, and labels within the dataset
DMP Overall data management strategy for the project lifecycle

File Formats: Open vs. Proprietary

For long-term data sharing and archiving, open file formats are strongly preferred:

Data Type Open Format (✅ Preferred) Proprietary Format
Tabular data .csv, .tsv .xlsx (Excel)
Text .txt, .md, .pdf/a .docx (Word)
Images .tiff, .png .psd (Photoshop)
Audio .wav, .flac .mp3
Geospatial .geojson, .shp .mdb (ArcGIS)

Open formats can be read by any software now and in the future — proprietary formats may become unreadable if the software is discontinued.

Data Publishing with Dryad

Dryad is a curated, open access data repository that makes research data freely available. Benefits include:

  • Assigns a DOI to your dataset — making it citable
  • Increases your citation count (data citations are recognized)
  • Facilitates reproducibility and research validation
  • Many journals require or recommend Dryad for supplementary data

Steps to Publish Data on Dryad

  1. Organize and document your data (README + codebook)
  2. Convert to open file formats where possible
  3. Create a Dryad account (free for UCR affiliates)
  4. Upload and describe your dataset
  5. Receive a DOI and share it in your publications
